## 3. Phase 9: Forecast Planner (Next)

**Objective**: Provide a simulation of Net Worth trajectory up to age 95.

### 9.1 Forecast Settings
**Acceptance Criteria**:
- [ ] **Birth Year**: User can enter birth year; system accurately calculates current and future age.
- [ ] **Rates**: User can set Inflation Rate and Investment Return Rate. Changes to these immediately affect the graph.

### 9.2 Base Cost of Living (CoL)
**Acceptance Criteria**:
- [ ] **Actuals Driven**: Checking a category (e.g., "Groceries") adds the sum of the *last 12 months* of that category to the Base CoL.
- [ ] **No Manual Override**: The Base CoL is strictly derived from the selected categories' actual data.

### 9.3 The Simulation (The Runway)
**Acceptance Criteria**:
- [ ] **Logic Accuracy**: Verify calculation: `Next_Year_NW = (Assets * (1 + Return)) + Income - Expenses`.
- [ ] **Drawdown Visualization**: If Expenses > (Income + Growth), the Net Worth line should visibly decline.
- [ ] **Compounding**: Verify that positive savings result in exponential growth (compounding returns).

### 9.4 The Graph
**Acceptance Criteria**:
- [ ] **X-Axis**: Years/Age (e.g., "2025 (Age 50)").
- [ ] **Y-Axis**: Net Worth (Currency).
- [ ] **Visibility**: The chart clearly shows if/when the Net Worth line crosses zero (exhaustion of assets).
