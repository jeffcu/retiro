# Curie Trust - Acceptance Test Plan v1.4

## 1. Introduction

This document outlines the acceptance criteria and testing strategy for each phase of the Curie Trust Financial Control Center project, as defined in the revised Project Requirements Specification (PRS), Section 10. The plan follows a user-feature-centric approach, verifying that each vertical slice of functionality is complete and correct before proceeding.

---

## 2. Phase 0-6: Core Functionality (Completed)

All acceptance criteria for phases 0 through 6 are considered stable.

---

## 3. Phase 7: Retirement & Estate Forecasting

**Objective**: Provide comprehensive net worth tracking (including Real Estate) and future-looking financial projections.

### 7.1 Real Estate Data Management

**Objective**: Verify the creation, update, and display of property assets.

**Acceptance Criteria**:
- [ ] **Create Property**: User can add a "Principal Residence" with Purchase Price, Mortgage, Current Value, and Appreciation Rate. It appears in the list.
- [ ] **Limit Check**: Verify logic allows adding additional properties.
- [ ] **Update Property**: Edit the "Current Value Estimate" of an existing property. The change is persisted.
- [ ] **Equity Calculation**: Verify that Equity is calculated as `Current Value - Mortgage Balance`.

### 7.2 Total Net Worth Hero

**Objective**: Verify the Home Screen Hero correctly aggregates Portfolio and Real Estate data.

**Acceptance Criteria**:
- [ ] **Display**: The Hero section now shows two values: "Liquid Portfolio" and "Total Net Worth (Inc. Real Estate)".
- [ ] **Calculation**: 
    - Liquid Portfolio = Total Market Value of Holdings.
    - Total Net Worth = Liquid Portfolio + Sum(Property Equity).
- [ ] **Verification**: Manually sum the portfolio value and property equity to ensure the displayed total is correct.

### 7.3 Future Income Streams (Completed)

**Acceptance Criteria**:
- [X] **CRUD**: User can create, read, and delete future income streams (Social Security, RMDs).
